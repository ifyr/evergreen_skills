In some cases, an engineer is expected to know about certain fields already, such as front-end, back-end, architecture, infrastructure, or security. The following cross-framework concepts and principles can serve as the technical foundation for those fields.

## Sub-Sections

| Section | Description |
|---------|-------------|
| [Front-End](2.8a-Front-End.md) | API communication, DOM, browser events, responsive design, CSR vs. SSR, state management, WebSocket |
| [Back-End](2.8b-Back-End.md) | API design, message brokers, relational/NoSQL databases, ORM, session handling, error handling/auditing/logging |
| [Architecture](2.8c-Architecture.md) | REST/SOAP, circuit breaker, everything as code, monolith vs. microservices, DDD, hexagonal architecture, data modeling, service mesh |
| [Infrastructure](2.8d-Infrastructure.md) | VMs vs. containers, processes vs. threads, IaaS/PaaS/SaaS, load balancing, observability |
| [Security](2.8e-Security.md) | IAM, public-key cryptosystems, TLS/SSL, least privilege, DoS/DDoS, SQL injection, XSS/CSRF |
