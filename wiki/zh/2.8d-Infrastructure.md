# 2.8d 基础设施

- **虚拟机 vs 容器**：VM 有独立 Guest OS，隔离强但资源开销大；容器共享宿主机内核，启动快，密度高。
- **进程 vs 线程**：进程是资源分配的基本单位（独立内存空间），线程是 CPU 调度的基本单位（共享进程内存）。进程隔离安全但通信成本高；线程通信轻量但共享内存容易引入并发 bug。
- **控制器-代理 / 主从模式**：常见分布式系统拓扑。
- **IaaS / PaaS / SaaS**：从下到上，管理职责逐渐从用户转移到云厂商——IaaS 你管 OS 以上，PaaS 你只管代码，SaaS 你只管使用。
- **Web 服务器 / 反向代理 / 负载均衡**：接收请求→分发到后端→返回响应。Nginx、HAProxy。
- **冗余设计**：消除单点故障——多副本、多区域、自动故障转移。
- **延迟（Latency）**：数据从源头到目的地的时间。关注 P50/P95/P99 而非平均值。
- **可观测性（Observability）**：超越传统监控（预设阈值和已知问题检测），让你**能对系统提出任意之前没想过的问题**而无需重新埋点。

  三大支柱：
  - **宽结构事件（Wide Structured Events）**：收集足够丰富的运行时上下文。
  - **迭代查询**：按任意维度分组、过滤、聚合。
  - **分布式追踪（Distributed Tracing）**：跟踪请求在分布式系统中的完整执行路径。

  目标：在问题影响用户之前发现；安全实验和优化；促进跨团队知识传递。
