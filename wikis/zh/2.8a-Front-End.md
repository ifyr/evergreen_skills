# 2.8a 前端开发

---

- **API 通信**：不同架构标准（REST、GraphQL）、数据格式（JSON、XML）、请求生命周期。
- **DOM**：文档对象模型——浏览器将 HTML 解析为可编程的节点树。虚拟 DOM 是框架（React）在内存中维护的轻量表示，通过 diff 算法最小化实际 DOM 操作。
- **浏览器事件**：捕获与冒泡阶段、事件委托、自定义事件。
- **响应式设计**：通过媒体查询、弹性布局（flexbox/grid）、相对单位（rem/vw）使界面适配不同屏幕尺寸。核心策略是**渐进增强**——先保证基础内容可用，再为高级浏览器添加增强体验。
- **CSR vs SSR**：
  - 客户端渲染（CSR）：浏览器下载 JS bundle 后在本地渲染。首屏慢，SEO 差，但交互流畅。
  - 服务端渲染（SSR）：服务器生成完整 HTML 返回。首屏快，SEO 好，但服务器负载高。
  - 混合方案（SSG / ISR）：构建时预渲染或增量静态再生，取两者之优。
- **分页**：大数据集的加载策略——偏移分页（offset）、游标分页（cursor）、无限滚动。
- **状态管理**：组件间共享状态的一致性问题。方案：Prop Drilling → Context → Redux / Zustand / Pinia。
- **MVC / MVP / MVVM**：将界面逻辑、业务逻辑和数据模型分离的架构模式。
- **WebSocket**：全双工通信协议，适合实时应用（聊天、协作编辑、实时数据推送）。
